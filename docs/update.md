# Обновления между версиями

Последняя версия ansible-роли совместима со всеми инсталляциями кластера, установленными до этой версии

Главные вехи роли, которые влияют на совместимость:
- 24.1   - первый выпуск роли с фиксированными именами инстансов и каталогов
- 25.1.2 - формирование файла `report.yml` по результатам работы роли. В дальнейшем, из этого файла загружается служебная информация о кластере
- 25.4.1 - автогенерация имен инстансов: на каждом сервере имена файлов и каталогов присваиваются инкрементально: `001`, `002`, `003` и т.д., фиксируются в конфигурационном файле
- 25.5.3 - убрана привязка по имени инстанса в конфигурационном файле, имя инстанса оставлено в виде комментария
- 26.6.1 - имена инстансов и каталогов присваиваются по именам bin-портов
- 26.6.2 - обновлен формат конфигурационного файла инстанса под пикодату версии 26; из конфигурационного файла убрано имя инстанса

# Обновление кластера Пикодата с 25.х на 26.х

> Внимание! Если нужно проводить работы без простоя работы кластера, то при выполнении операций рестарта и обновлении конфига используйте параметр `--limit <имя сервера>`

- Обновите роль до последней версии (26.6.2)
- Обновите кластер до версии 25.5.9, используя тег `install_pkgs`, затем `restart`
- Обновите кластер до версии 26.6.1, используя тег `install_pkgs`
- Выполните роль с тегом `deploy` для обновления конфигурационных файлов, по окончании роли инстансы кластера перезапустятся!

> Внимание! Переход на мажорную версию 26 возможен только с версии 25.5.9!
